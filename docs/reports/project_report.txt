# SHOPPING CART SYSTEM PROJECT REPORT

## CHAPTER 1: INTRODUCTION

### 1.1. Introduction about Project

In today's digital era, e-commerce has become an integral part of our daily lives. The Shopping Cart application aims to provide a seamless online shopping experience by offering a centralized platform where customers can easily browse, compare, and purchase products from a wide range of categories, all in one place.

For customers, the Shopping Cart application offers an intuitive and user-friendly interface that makes it easy to explore different products, view detailed information, and make purchases with just a few clicks. With detailed descriptions, product images, and transparent pricing, customers can make informed decisions and complete their shopping with confidence.

On the other hand, the Shopping Cart application empowers administrators with powerful tools to manage products, inventory, orders, and user interactions efficiently. From adding and editing product listings to monitoring orders and processing payments, administrators have full control over the platform's offerings and operations. With real-time insights and reporting capabilities, administrators can make data-driven decisions to optimize the platform and enhance the user experience.

This project implements a comprehensive e-commerce shopping cart system that allows users to browse products, add items to cart, and complete purchases. The system is built using Java Servlets and JSP technology with a MySQL backend database.

### 1.2. Need of Computerization of System

#### Streamline Shopping Process
The primary objective of the system is to streamline the shopping process for customers, allowing them to easily browse, compare, and purchase products from a diverse selection of categories and items.
This is achieved through a user-friendly interface which is easy to navigate from one page to another, with intuitive product searching, filtering, and easy checkout processes.

#### Customer Management
Another objective is managing customer details, their orders, and their queries. This is achieved by implementing a Customer Management System within the application, enabling administrators to track customer interactions, manage inquiries, and personalize communication.

#### Efficient Inventory Management
The system enables efficient inventory management, including tracking product availability, updating stock levels, and managing product information. This is achieved through a comprehensive admin interface that allows for real-time inventory updates and product management.

#### Payment Processing
The system streamlines the checkout process for customers and manages payments securely. This is achieved through an integrated payment processing system that allows customers to complete purchases, make payments, and receive order confirmations instantly.

Additionally, the system aims to:
- Provide a centralized platform for product management
- Enable secure user authentication and authorization
- Facilitate efficient order processing and management
- Offer real-time inventory tracking

### 2.1. Feasibility Study of Software and Its Types

#### Java
Java is a robust, platform-independent programming language widely used for developing enterprise-level applications. Its "write once, run anywhere" capability makes it an excellent choice for web applications that need to run across different platforms. Some key aspects of Java include:

1. **Platform Independence**: Java applications can run on any device or operating system that has a Java Virtual Machine (JVM), making it highly portable and flexible.

2. **Features**: Java offers a rich set of features including object-oriented programming, exception handling, multithreading, garbage collection, and an extensive standard library, all of which contribute to building robust and scalable applications.

3. **Enterprise Support**: Java Enterprise Edition (Java EE) provides a set of specifications and APIs for developing enterprise applications, including servlets, JSP, JDBC, and more, which are used in the Shopping Cart application.

4. **Community and Support**: Java has a large and active community, extensive documentation, and a wide range of libraries and frameworks, making it easier to find solutions to development challenges.

#### MySQL
MySQL is a popular open-source relational database management system known for its reliability, performance, and ease of use. Some key aspects of MySQL include:

1. **Reliability**: MySQL is known for its stability and has been widely used in production environments for many years.

2. **Performance**: MySQL offers good performance for web applications, with features like caching, indexing, and query optimization.

3. **Scalability**: MySQL can handle large amounts of data and high traffic loads, making it suitable for growing applications.

4. **Integration**: MySQL integrates well with many programming languages and frameworks, including Java, which is used in the Shopping Cart application.

## CHAPTER 4: TESTING AND IMPLEMENTATION

### 4.1. Testing Methodology

Testing methodologies in Software Testing are strategies designed to tackle one of the biggest problems in development: finding errors and fixing them. With the growing number of technologies, coding languages, possible integrations, and more, it's important to have appropriate testing methodologies for different development processes.

### 4.2. Unit Testing

Unit testing is the process of testing individual units or components of the Shopping Cart application to ensure they work properly in isolation. Here is how unit testing works for this application:

1. **Identify Units**: The Shopping Cart application is broken down into different units or components, such as user authentication, product management, cart operations, order processing, etc.

2. **Write Test Cases**: Test cases are written for each unit to cover different scenarios and use cases, such as adding a new product, updating cart items, handling errors gracefully, etc.

3. **Set up a Test Environment**: A test environment is established where unit tests can run independently from the rest of the system, using test doubles to simulate interaction with other parts of the system when necessary.

4. **Run Tests Separately**: Unit tests are run separately for each component to ensure that any errors or issues are specific to the unit being tested, making it easier to identify and fix problems.

5. **Evaluate Results**: Test results are evaluated to identify any errors or unexpected behavior. If a test fails, the cause is investigated and code changes are made to fix the problem.

6. **Iterate and Refactor**: As problems are discovered and fixed, unit tests are updated accordingly to maintain accurate test coverage.

7. **Automate Tests**: Unit tests are automated to enable frequent execution and integration into the development process, helping to catch regressions early and ensure that new changes don't disrupt existing functionality.

### 4.3. Module Testing

Module testing, also called component testing, is the process of testing individual modules (units of code) within the Shopping Cart application. The goal is to ensure that each module performs as expected by testing its input and output, boundary conditions, and behavior under various scenarios.

By testing each component separately, developers can identify and fix problems in individual components before they are integrated into the larger system, making it easier to isolate and debug issues and resulting in a more reliable and robust application.

### 4.4. System Testing

System testing involves testing the integrated Shopping Cart application as a whole to ensure it meets requirements and functions as intended in its target environment. Rather than testing specific parts of the system, system testing evaluates the overall functionality, performance, reliability, and security.

This process verifies that the application performs as expected and meets user needs by conducting both functional testing (to ensure features work correctly) and non-functional testing (to assess aspects like scalability, performance, and usability).

### 4.5. Alpha/Beta Testing

Alpha and beta testing are two stages of user acceptance testing (UAT) involving real users:

**Alpha Testing**: Conducted by the development team in a controlled environment to identify defects, usability issues, or other problems before releasing the application to external users. Alpha testing focuses on overall quality, performance, and functionality.

**Beta Testing**: Conducted by a small group of external users in a real-world environment to gather feedback on performance, usability, bugs, and potential improvements before public release. Beta testers provide valuable insights from an end-user perspective.

### 4.6. White Box/Black Box Testing

**White-box Testing**: This approach, also known as structural or glass-box testing, involves examining the application's internal logic and structure. Test cases are based on an understanding of the code, including paths, branches, and conditions. Testers have access to the source code and use techniques like path testing, code coverage analysis, and statement coverage to ensure comprehensive testing.

**Black-box Testing**: This approach, also known as functional testing, focuses on testing the application's external behavior without knowledge of its internal implementation. Test cases are derived from requirements, features, and specifications. Testers focus on inputs and outputs, boundary conditions, and expected behaviors without access to the source code.

### 4.7. Implementation

Implementing the Shopping Cart application involves several key steps:

1. **Requirement Acquisition**: Document functional and non-functional requirements, including features, user roles, security requirements, and performance expectations.

2. **System Design**: Design the architecture and data model based on the gathered requirements.

3. **Development**: Develop the application according to the specified requirements and design.

4. **Testing**: Conduct unit testing, integration testing, and system testing to ensure the application functions correctly.

5. **Deployment**: Prepare the application for deployment, configure servers, databases, and other infrastructure components.

6. **Maintenance and Support**: Monitor performance and stability, address bugs and enhancement requests post-deployment.

### 4.8. Post Implementation

**Monitoring and Evaluation**: Track system metrics like response time, uptime, and resource usage to evaluate performance.

**Support and Maintenance**: Provide technical support to administrators and users, establish processes for reporting and resolving issues.

**User Training and Adoption**: Offer training resources to help users effectively use the system, promote adoption through incentives and communication.

**Data Management and Backup**: Implement data management policies to ensure data availability, confidentiality, and integrity, perform regular backups.

**Security and Compliance**: Regularly assess the application's security posture and implement measures to mitigate threats.

**Continuous Improvement**: Establish processes for gathering and analyzing feedback from users and system metrics, prioritize improvements based on insights.

```sql
-- Database: `shopping-cart`

-- Table structure for table `product`
CREATE TABLE IF NOT EXISTS `shopping-cart`.`product` (
  `pid` VARCHAR(45) NOT NULL,
  `pname` VARCHAR(100) NULL DEFAULT NULL,
  `ptype` VARCHAR(20) NULL DEFAULT NULL,
  `pinfo` VARCHAR(350) NULL DEFAULT NULL,
  `pprice` DECIMAL(12,2) NULL DEFAULT NULL,
  `pquantity` INT NULL DEFAULT NULL,
  `image` LONGBLOB NULL DEFAULT NULL,
  PRIMARY KEY (`pid`)
);

-- Table structure for table `user`
CREATE TABLE IF NOT EXISTS `shopping-cart`.`user` (
  `email` VARCHAR(60) NOT NULL,
  `name` VARCHAR(30) NULL DEFAULT NULL,
  `mobile` BIGINT NULL DEFAULT NULL,
  `address` VARCHAR(250) NULL DEFAULT NULL,
  `pincode` INT NULL DEFAULT NULL,
  `password` VARCHAR(20) NULL DEFAULT NULL,
  PRIMARY KEY (`email`)
);

-- Table structure for table `orders`
CREATE TABLE IF NOT EXISTS `shopping-cart`.`orders` (
  `orderid` VARCHAR(45) NOT NULL,
  `prodid` VARCHAR(45) NOT NULL,
  `quantity` INT NULL DEFAULT NULL,
  `amount` DECIMAL(10,2) NULL DEFAULT NULL,
  `shipped` INT NOT NULL DEFAULT 0,
  PRIMARY KEY (`orderid`, `prodid`),
  INDEX `productid_idx` (`prodid` ASC) VISIBLE,
  CONSTRAINT `productid`
    FOREIGN KEY (`prodid`)
    REFERENCES `shopping-cart`.`product` (`pid`)
);

-- Table structure for table `transactions`
CREATE TABLE IF NOT EXISTS `shopping-cart`.`transactions` (
  `transid` VARCHAR(45) NOT NULL,
  `username` VARCHAR(60) NULL DEFAULT NULL,
  `time` DATETIME NULL DEFAULT NULL,
  `amount` DECIMAL(10,2) NULL DEFAULT NULL,
  PRIMARY KEY (`transid`),
  INDEX `truserid_idx` (`username` ASC) VISIBLE,
  CONSTRAINT `truserid`
    FOREIGN KEY (`username`)
    REFERENCES `shopping-cart`.`user` (`email`),
  CONSTRAINT `transorderid`
    FOREIGN KEY (`transid`)
    REFERENCES `shopping-cart`.`orders` (`orderid`)
);

-- Table structure for table `usercart`
CREATE TABLE IF NOT EXISTS `shopping-cart`.`usercart` (
  `username` VARCHAR(60) NOT NULL,
  `prodid` VARCHAR(45) NOT NULL,
  `quantity` INT NULL DEFAULT NULL,
  PRIMARY KEY (`username`, `prodid`),
  INDEX `productid_idx` (`prodid` ASC) VISIBLE,
  CONSTRAINT `username`
    FOREIGN KEY (`username`)
    REFERENCES `shopping-cart`.`user` (`email`),
  CONSTRAINT `prodid`
    FOREIGN KEY (`prodid`)
    REFERENCES `shopping-cart`.`product` (`pid`)
);

-- Table structure for table `user_demand`
CREATE TABLE IF NOT EXISTS `shopping-cart`.`user_demand` (
  `username` VARCHAR(60) NOT NULL,
  `prodid` VARCHAR(45) NOT NULL,
  `quantity` INT NULL DEFAULT NULL,
  PRIMARY KEY (`username`, `prodid`),
  INDEX `demandproductid_idx` (`prodid` ASC) VISIBLE,
  CONSTRAINT `demanduserid`
    FOREIGN KEY (`username`)
    REFERENCES `shopping-cart`.`user` (`email`),
  CONSTRAINT `demandproductid`
    FOREIGN KEY (`prodid`)
    REFERENCES `shopping-cart`.`product` (`pid`)
);
```
